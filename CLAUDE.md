# CLAUDE.md

## Project defaults
- 답변은 한국어로 작성한다.
- 변경 제안 시 항상 영향 범위를 먼저 설명한다.
- 추측하지 말고, 확인한 사실과 추정을 구분한다.
- 큰 변경 전에는 계획을 먼저 제시한다.

## Engineering defaults
- 기존 아키텍처/네이밍/레이어를 우선 존중한다.
- 불필요한 리팩터링은 제안하지 않는다.
- 테스트 가능하면 관련 테스트 관점도 함께 본다.

- 학부모는 이용금액, 이용정지일 수정 금지
- 목록 조회 시 inactive 학생 기본 제외
- 모바일에서 숫자 입력은 가능하면 숫자 키패드 유도
## SweetAlert2는 간단 입력에만 사용, 긴 폼은 페이지/드로어 우선 검토
