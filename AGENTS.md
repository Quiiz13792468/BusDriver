# AGENTS.md

## 프로젝트 개요
이 프로젝트는 통학버스 관리 시스템이다. (BusDriver v2 전면 재구축)

주 사용자:
- DRIVER: 버스 기사 (50~80대) — 핵심 사용자, 전체 기능 접근
- PARENT: 학부모 (50~70대) — 자녀 정보 조회 및 이의제기만 가능
- ADMIN: 서비스 운영자 (개발자 본인) — Phase 2에서 별도 관리 화면 구성

핵심 목표:
- 최소 클릭으로 사용 가능
- 모바일/태블릿 환경에서도 쉽게 사용
- IT에 익숙하지 않은 사용자도 사용 가능

---

## 기술 스택

- Framework: Next.js 15 (App Router)
- Language: TypeScript
- UI: shadcn/ui + Tailwind CSS
- Backend: Supabase (Auth + DB)
- Email: Resend (인증·알림 전용, 무료 3,000건/월)
- 광고: Google AdSense (모바일 하단 배너 + 데스크톱 사이드바)
- 설치: PWA (1단계) → Capacitor iOS/Android 앱스토어 배포 (2단계)

### DRIVER 전용 글로벌 액션 (헤더 영역)
어느 탭에서든 접근 가능:
- **입금 등록** 버튼
- **주유 등록** 버튼

---

### DRIVER — 하단 탭바 4탭

#### 1. 홈 (`/dashboard`)
| 섹션 | 내용 |
|------|------|
| 오늘 입금 예정 | 오늘이 입금일인 학생 리스트 |
| 미납 알림 | 입금일 지났으나 미납인 학생 리스트 |
| 입금확인 요청 | 학부모가 보낸 확인 요청 알림 리스트 |
| 이번달 입금 요약 | 총 입금 현황 → 금액 클릭 시 날짜/학생/금액 상세 |

#### 2. 학생관리 (`/schools`)
- 조회 조건: 전체 학생 (기본값) / 학교별 필터
- 학생 등록: 목록 화면 우측 상단 버튼
- 학생 클릭 → 상세 페이지
  - 상세 정보: 학생 이름·전화번호, 학부모 이름·전화번호, 입금일, 입금액, 이용타입(등교/하교/등하교), 시작일, 종료일
  - 편집 가능
  - 월별 입금 내역 버튼 → 해당 학생 입금 이력 조회

#### 3. 장부 (`/payments`)
- 전체 학생 통합 입금 이력 + 학생별 조회
- 월별 매출 (해당 월 총 입금액)
- 주유 내역 (날짜·금액·메모, DRIVER 직접 입력)

#### 4. 게시판 (`/board`)
- 공지 작성 (DRIVER만 가능)
- 학부모와 1:1 채팅 (iMessage 스타일)
- 답장 기능: 메시지 꾹 누르기 → 원본 미리보기 포함 답장 → 클릭 시 원본 스크롤 포커스 (WhatsApp 방식)
- 조회 조건: 공지 / 학교 선택

---

### PARENT — 하단 탭바 3탭

#### 1. 홈 (`/dashboard`)
- 자녀 카드 스와이프 (자녀 여러 명)
- 이번달 납부 현황 요약, 미납금 강조
- 입금 등록: 글로벌 헤더 버튼
- 수정 요청·확정 액션 (홈 알림에서 처리)
- 알림: 버스기사 입금 등록 알림, 게시판 답글 알림

#### 2. 납부내역 (`/payments`)
테이블 구조:
- 열: 월(틀고정) / 첫째 / 둘째 / 셋째 ... (가로 스크롤)
- 행: 연도별 조회 (최초 가입 연도부터)
- 셀 클릭 → 팝업 상세:

```
┌─────────────────┬─────────────────┐
│ 입금일          │ 금액            │
├─────────────────┼─────────────────┤
│ 등록일          │ 최초 등록자     │
├─────────────────┴─────────────────┤
│ 메모                              │
├─────────────────┬─────────────────┤
│ [수정 요청]🔴   │ [확정]🟢        │
└─────────────────┴─────────────────┘
```

버튼 색상:
- `[수정 요청]` → `#FF3B30` (Danger)
- `[확정]` → `#34C759` (Success)

#### 3. 게시판 (`/board`)
- 버스기사와 1:1 채팅 (iMessage 스타일)
- 메시지마다 자녀 태그 표시
- 메시지 검색 기능
- 공지 상단 고정 배너
- 답장 기능 (DRIVER 게시판과 동일)

## 로그인 화면

```
┌──────────────────────────────┐
│  [버스기사]    [학부모]       │ ← 역할 선택
│                              │
│  아이디  [________________]  │
│  비밀번호[________________]  │
│                              │
│         [로그인]             │
│                              │
│  아이디 찾기 | 비밀번호 찾기  │
└──────────────────────────────┘
```

| 항목 | 내용 |
|------|------|
| 역할 기본값 | 최초: DRIVER / 이후: 마지막 로그인 역할 자동 선택 |
| 아이디·비밀번호 저장 | 체크박스 선택 시 기기 로컬 저장 |
| 자동 로그인 | Supabase 세션 유지 |
| 회원가입 | 초대 링크 경유만 가능 (로그인 화면에 가입 버튼 없음) |

### 로그인 아이디 구조
- 가입 시 입력: 실제 이메일 + 로그인 아이디 + 비밀번호
- 로그인: 로그인 아이디 + 비밀번호
- 실제 이메일: 비밀번호 찾기·포인트 인증 전용 (화면 노출 최소화)

## 입금 확인 프로세스 (양방향 합의)

누가 먼저 등록하든 동일한 프로세스:

```
[DRIVER or PARENT] 입금 등록
        ↓ 알림
[상대방] 확인
    ├─ [확정] → 최종 확정 ✅
    └─ [수정 요청] + 메모 → 등록자에게 알림
              ↓
         [등록자] 금액 수정 후 [확인 요청]
              ↓ 알림
         [상대방] 재확인 → 확정될 때까지 반복
```

### payments 상태값
- `PENDING`: 검토 중 (등록 후 상대방 미확인)
- `DISPUTED`: 수정 요청 중
- `CONFIRMED`: 양방 최종 확정

## 핵심 도메인 규칙

### 입금 처리
- 기본 이용금액 우선순위: 학교별 고정금액 > 학생별 고정값
- 입금은 누적 계산 기준
- 부족금액 = 이용금액 - 총 입금액

### 이용 정지
- 이용 정지 학생은 조회 리스트에서 기본 제외
- 삭제하지 않고 inactive 상태 유지

### 초대 링크
- 유효기간: 최대 7일
- 사용 횟수: 1회 (1명만 가입 가능)
- 역할: 생성 시 DRIVER or PARENT 선택

### v2에서 제거된 기능
- 노선/정차지점 관리 (DRIVER, PARENT 모두)
- 탑승지점 변경

## 데이터 설계 가이드

### 인증/사용자
- `auth.users`: Supabase Auth 기본 사용자
- `profiles`: role(ADMIN|DRIVER|PARENT), 로그인 아이디, auth.users 1:1 연결
- `invite_tokens`: 역할, 유효기간(7일), 사용 여부, 1회 사용

### 학교/학생
- `schools`: 학교 정보, 기본 월 이용금액
- `students`: 학생 이름·전화번호, 학부모 이름·전화번호, 학교 연결, 학부모 계정 연결, 이용타입(등교/하교/등하교), 시작일, 종료일, 입금일, 입금액, is_active

### 입금/주유
- `payments`: 상태(PENDING|DISPUTED|CONFIRMED), 금액, 입금일, 등록일, 최초 등록자(DRIVER|PARENT), 메모
- `fuel_records`: 날짜, 금액, 메모, 등록자(DRIVER)

### 게시판/알림
- `board_posts`: 공지 (DRIVER 작성, 학교 대상)
- `board_messages`: 1:1 채팅 메시지, 자녀 태그, 답장 연결(reply_to)
- `notifications`: 입금 확인 요청, 수정 요청, 게시판 답글 알림, 읽음 여부

### 감사 로그
- `auth_login_audit`: 로그인 이력 (역할, 성공 여부, 시각)
