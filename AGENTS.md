* Next.js, SweetAlert2, Typescript 사용
* Superbase 사용
* 2025년 11월 기준 최신 트렌드에 맞는 디자인 색감, 스타일 사용
* 주된 사용자 타겟은 50~70대 성인
* 모든 컨텐츠는 직관적이고 사용하기 쉽게 구성해야한다



* 프로젝트 설명



* \[공통기능]
* 1\. 로그인
* &nbsp; - 관리자(버스 기사), 학부모 로그인이 구분되어야함
* 
* 2\. 자동 로그인 로그인 이력이 있으면 자동 로그인
* 
* 3\. 로그인 유지시간(1일)
* 
* 4\. 로그아웃(30분 뒤 자동 로그아웃)
* 
* \[관리자 전용 기능]
* 0\. 대시보드
* &nbsp; - 학생별 입금 등록/수정/삭제 팝업으로 최대한 간편하게 등록할 수 있어야함
* &nbsp;   1) 학생 검색 및 선택, 입금유형(라디오버튼이며 일반입금(기본선택), 부분입금으로 선택 가능해야함), 입금액(기본 입금액은 학생정보-이용금액), 비고, 저장버튼, 닫기버튼
* 
* 1\. 학교 관리
* &nbsp; - 학교 추가/수정/삭제
* &nbsp; - 학교별 학생 매핑 등록
* 
* 2\. 학생 관리(추가/수정)
* &nbsp; - 학생 정보를 입력/수정 가능
* &nbsp; - 입력정보 목록
* &nbsp;   1) 학생 이름
* &nbsp;   2) 학생 전화번호
* &nbsp;   3) 학부모 이름
* &nbsp;   4) 학부모 전화번호
* &nbsp;   5) 이용금액
* &nbsp;   6) 탑승위치
* &nbsp;   7) 메모
* &nbsp;   8) 이용 정지일(=관리자가 날짜를 입력하는 순간부터 기타 조회화면에서 안나옴)
* 
* 3\. 노선 관리
* &nbsp; - 버스 이동 노선을 등록한다
* &nbsp; - 등록한 노선은 학교에 매핑 등록
* &nbsp; - 노선에 정차지점을 등록한다. 사용방법은 아래와 같다.
* &nbsp;   1) 네이버 맵, 카카오 맵(둘중에 api를 사용하기 편한걸로 아무거나 상관없음)에 마커를 찍는다.
* &nbsp;   2) 마커에 위치명, 설명을 입력 후 저장
* 
* 4\. 입금관리
* 
* 5\. 실적 관리(아래의 기능들이 검색 및 조회 가능해야함)
* &nbsp; - 월별, 년별 총 실적 조회
* &nbsp; - 학교별 월별 실적 조회
* &nbsp; - 학생별 월별 실적 조회
* &nbsp;   1) 실적에 포함되어야할 필수 칼럼(학생이름, 학교, 입금금액, 부족금액, 비고)
* &nbsp;   2) 월 클릭시 서브 팝업으로 입금이력 조회가능
* 
* 6\. 게시판 관리
* &nbsp; - 게시글 등록
* &nbsp; - 댓글, 대댓글 등록
* &nbsp; - 관리자가 글을 등록하면 학부모들은 게시판에서 마찬가지로 확인 및 댓글 작성 가능
* 
* \[학부모 전용 기능]
* 0\. 대시보드
* &nbsp; - 학부모별 학생 정보 확인/수정 가능(이용금액, 이용 정지일은 수정불가)
* &nbsp; - (년도 선택) 월별 입금내역 현황(해당월 마다 입금 확인요청 버튼이 있으며 클릭시 관리자에게 알람이 간다)
* 
* 1\. 노선확인
* &nbsp; - 학생별 학교에 등록된 노선 확인가능
* 
* 2\. 게시판
* &nbsp; - 관리자가 작성한 게시글 확인/댓글작성 가능
* &nbsp; - 학부모가 관리자에게 Q\&A처럼 게시글 작성 가능
